Abstract

The ROLE Responsive Open Learning Environments (ROLE) is a European collaborative project with 16 internationally renowned research groups from 6 EU countries and China. ROLE´s main task is to deliver and test prototypes of highly responsive Personal Learning Environments (PLE), offering breakthrough levels of effectiveness, flexibility, user-control and mass-individualisation. This document is the specification of the ROLE ontology describing the classes and properties that compose it.

1 Introduction

The ROLE ontology is meant to serve as basis to perform reasonning on repositories of tools and learning resources used in the context of self-directed learning. It describe both tools and learning processes from a Technology-enhanced learning perspective. New classes and properties have therefore been introduced to precisely account for the specificity of this application domain, but we also took care to reuse and anchor our classes and properties to existing ontologies (such as DCTerms, FOAF, SIOC) whenever this was possible. It has been designed to fulfill the need of the ROLE project in terms of meta-modelling and description of learning tools and processes.

For example, the ROLE Widget Store is a repository for all ROLE widgets that can be selected and used within a personal learning environment or LMS. The ROLE widget store offers the filtering, recommendation and search of widgets that fit to a specific need of a user. The widget store is also open to widgets from many providers; therefore, it offers functionalities to manage widgets from different vendors or partners of ROLE. The ROLE ontology provides the concepts and properties to precisely describe each tool in relation to, e.g., the learning domain or the learning-wise functionalities. The goal in this respects is to expose the descriptions of the tools and allow third-party agents to exploit ROLE catalogue of widgets.

Another innovation proposed by the ROLE framework is th ability to organize documents, tools, and other resources in context-relevant bundles of learning resources. These bundle can be shared to the communities in online catalog like the ROLE Widget Store. These bundle can then instantiated in web spaces within a space-enabled container (Moodle, Graaasp, etc.). A space consists in a bundle of resources, participants, and tools. For example, students can create a space dedicated to one course where they can invite classmates, the teacher, etc., and collect and organize relevant resources and applications by linking them to the space. Spaces, Resource bundles and their properties can be precisely described with ROLE ontology.

3. ROLE Ontology overview and examples

3.1. Overview

This following figure gives an overview of the classes and properties defined in the ROLE ontology. The model is divided into three colors:

Core ontology - orange boxes
Used to define a context for learning in the form of bundles containing tools and learning resources. A bundle is a recipe which can be instantiated at any time into a concrete space where also persons are associated with the space as owners or members.
Learning ontology - blue boxes
Centered on Learning activities, Learning strategies, and Functionalities. The Learning strategies organizes several learning activities and functionalities provides a bridge to the use of tools.
Competence ontology - green boxes
Provides a way to describe comptencies, both those already aquired and those planned in the form of goals.

3.2. Example

Here is example describing a Widget registered at the ROLE Widget Store:

<role:Widget rdf:about="http://www.role-widgetstore.eu/content/xmpp-multiuser-chat">
    <dcterms:source rdf:resource="http://role-is.dbis.rwth-aachen.de/gadgets/chat/xmpp-muc-gadget.xml"/>
    <dcterms:title xml:lang="en-US">XMPP Multiuser Chat </dcterms:title>
    <dcterms:description xml:lang="en-US">An XMPP Multiuser Chat Gadget</dcterms:description> 
    <foaf:maker>
        <foaf:Person rdf:about="http://www.role-widgetstore.eu/semantices/person/daniel">
            <foaf:name>daniel</foaf:name>
            <foaf:mbox>daniel@role-project.eu</foaf:mbox>
    	</foaf:Person>
    </foaf:maker>  
    <foaf:img>default screenshot of the Tool</foaf:img>
    <foaf:depiction>thumbnail of the Tool</foaf:depiction> 
    <role:category rdf:resource="http://dbpedia.org/page/Category:Communication"/>
    <role:toolFunctionality rdf:resource="http://purl.org/role/terms/FunctionalityTF1" />
</role:Widget>

Class: role:Configuration

URI: http://purl.org/role/terms/Configuration

Configuration - When a resource is to be added to a context, the resource often need to be contextualized or configured. That means providing additional information on this resource that is specific to this context. Making statements on the original resource would in the worst case affect other uses of the same resource. To reach the resource that is being contextualized/configured use the role:configures property. It is up to each subclass to specify if and how properties on the original resource should be copied, provided as a possible override, or always be looked up on the original resource.

Class: role:W3CWidget

W3C Widget - W3C Widgets are full-fledged client-side applications that are authored using Web standards such as HTML and packaged for distribution.
