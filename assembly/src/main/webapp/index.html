Responsive Open Learning Environments (ROLE) SDK

Version: ${project.version}

The ROLE Software Development Kit (SDK) was designed as a set of tools to support developers in their implementation of software components for the establishment of Responsive Open Learning Environments (ROLEs). The SDK is mainly powered by a development server already equipped with a set of services, libraries and gadgets developed in the context of the EU FP7 IP ROLE as well as hosting documentation to get you kick-started for your own developments.

Contents

Once the SDK Development Server was started successfully (see instructions below), the SDK and all of its contents (including the ones you add by yourself, which is useful during development) become available at http://127.0.0.1:8073/role/. A list of reachable code and documentation components is given below.

Code

  • ROLE Java Web Application Archives - the set of ROLE Web services & applications powering the SDK Development Server.
  • ROLE JavaScript libraries - a set of JavaScript libraries to support the development of new widgets or complete ROLEs.
  • ROLE Sample Widgets - a set of sample widgets demonstrating the use of ROLE Web services & libraries.
  • ROLE Development Widgets - a set of widgets supporting development of new widgets.

Documentation

  • ROLE Library API Documentation - API documentation for ROLE JavaScript libraries.

SDK Development Server

The ROLE SDK Development Server provides a basic infrastructure including ROLE software for your development purposes. The foundation for this infrastructure is a Jetty server hosting an Apache Shindig OpenSocial container. The server hosts a list of ROLE software components in terms of Web services, libraries, and gadgets supporting your own development. Furthermore, it includes an interactive testing environment, in which you can combine your own widget developments with already existing other widgets in so called widget spaces.

How to run the Development Server

  1. Start the server using ${SDK_HOME}/bin/start.sh (Linux, MacOS) resp. ${SDK_HOME}/bin/start.cmd (Windows).
  2. Point your browser to http://127.0.0.1:8073/ to access the ROLE SDK interactive testing environment.
    1. Log on with username testuser and password roleabdc OR authenticate via Google.
    2. Create your own ROLE Spaces.
    3. Try out your own ROLE widgets and combine them with already existing ones.

Development Server XMPP Configuration

Some real-time communication and collaboration features such as Chat or the ROLE Interwidget Communication Framework available with our development server require the connection to a pre-configured XMPP server. To configure your installation of the ROLE SDK Development Server for use with such a pre-configured XMPP server, the following fields should be added to the startup configuration in ${SDK_HOME}/bin/start.sh (Linux, MacOS) resp. ${SDK_HOME}/bin/start.cmd (Windows) as additional parameters given with -D<PARAM_NAME>=<PARAM_VALUE>. Notice that if no hostname to a pre-configured XMPP server is specified, the development server will still function, but not support chat and remote interwidget communication. A list of XMPP startup parameters is given below. Parameters only need to be specified, if different from default. A tutorial on how to pre-configure an XMPP server suitable for usage with the ROLE SDK Development Server is available from the ROLE Developer Wiki on Sourceforge. Additional parameters for client-side XMPP connection techniques are configurable in webapps/role-uu-prototype/script/config.js.
  • xmpp.host - Hostname of the XMPP server to connect
  • xmpp.port - Native port of the XMPP server to connect (default: 5222)
  • xmpp.user - Username of the ROLE Realtime Service (default: role-realtime-service)
  • xmpp.pass - Password of the ROLE Realtime Service (default: 807343V3R)
  • xmpp.mucs - Subdomain prefix of XMPP Multi-User Chat service to be used (default: conference)
  • xmpp.pubs - Subdomain prefix of XMPP Publish-Subscribe service to be used (default: pubsub)
